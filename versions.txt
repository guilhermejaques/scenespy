macOS compatibility summary

Current build/test Mac:
- macOS: 11.0.1
- Build: 20B29
- Architecture: x86_64 / Intel

Recommended distribution target:
- macOS 13 Ventura or later

Reason:
- The main Scenespy.app executable declares macOS 10.13 minimum.
- The bundled/private Python runtime also declares macOS 10.9/10.13 minimum depending on binary.
- PyTorch installed by install_runtime is compatible with macOS 10.13+ at the binary level.
- However, the installed MediaPipe binary declares macOS 13.0 minimum.
- Because Detect faces depends on the AI runtime, macOS 13+ is the safest public compatibility claim.

Architecture note:
- The current app build is x86_64 only.
- It runs natively on Intel Macs.
- On Apple Silicon Macs (M1/M2/M3/M4), it requires Rosetta 2.

Suggested release note:
Requires macOS 13 or later. Apple Silicon Macs require Rosetta 2.


Windows compatibility summary

Current build/test Windows:
- Windows: Windows 10
- Version: 10.0.19045
- Architecture: AMD64 / x64
- Python: 3.11.9

Recommended distribution target:
- Windows 10 64-bit version 19045 or later
- Windows 11 64-bit

Reason:
- The Windows release is built as Scenespy-windows-x64 and ships a x64 executable.
- install_runtime_windows.ps1 installs a private Python 3.11.9 under %LOCALAPPDATA%/Scenespy/python311/.
- Detect faces installs torch 2.5.1 and torchvision 0.20.1 from the official PyTorch CPU or CUDA 12.1 indexes.
- PyTorch on Windows requires the Microsoft Visual C++ Redistributable; install_runtime.py now checks for vcruntime140.dll, vcruntime140_1.dll, and msvcp140.dll and installs vc_redist.x64.exe when they are missing.
- FFmpeg/FFprobe are installed from the Gyan.dev essentials Windows build into %LOCALAPPDATA%/Scenespy/runtime/.
- CUDA mode is only selected automatically when nvidia-smi is available and the NVIDIA driver is new enough for the CUDA 12.1 PyTorch wheels.

Architecture note:
- The current Windows distribution is x64 only.
- 32-bit Windows and ARM64 Windows are not supported by this build.
- On machines without a supported NVIDIA driver, Detect faces falls back to CPU PyTorch.

Suggested release note:
Requires Windows 10 64-bit version 19045 or later, or Windows 11 64-bit. Detect faces may install the Microsoft Visual C++ Redistributable and uses CPU unless a supported NVIDIA CUDA setup is detected.


Linux compatibility summary

Current build/test Linux:
- Ubuntu latest release tested by the project owner.
- Result: build/runtime tests worked well on that Ubuntu version.
- The release script produces Scenespy-linux-x64 on the Linux machine where scripts/build_linux.sh is run.

Recommended distribution target:
- Ubuntu latest release, x86_64
- Other x86_64 glibc-based Linux distributions are best-effort until tested
- Build on the oldest Linux/glibc version that you want to support
- Practical public target: current Ubuntu LTS/Fedora/Arch-class systems with apt, dnf, or pacman available for FFmpeg installation

Reason:
- The Linux release is named Scenespy-linux-x64 and is generated by PyInstaller on the host Linux system.
- PyInstaller Linux binaries are not a universal cross-distro format; glibc and system library compatibility depend on the build host.
- Ubuntu latest is the only Linux distribution currently confirmed by project testing.
- The build Python must include tkinter support. On Debian/Ubuntu this usually means python3-tk or python3.11-tk.
- install_runtime.sh installs private Python 3.11.9 from python-build-standalone using the x86_64-unknown-linux-gnu target for x64 Linux.
- Detect faces installs torch 2.5.1 and torchvision 0.20.1 from the official PyTorch CPU or CUDA 12.1 indexes.
- FFmpeg/FFprobe are installed through apt, dnf, or pacman when they are not already available.
- CUDA mode is only selected automatically when nvidia-smi is available and the NVIDIA driver is new enough for the CUDA 12.1 PyTorch wheels.

Architecture note:
- The release folder name and build script describe the public Linux package as x64.
- install_runtime.sh contains private-Python targets for both x86_64 and aarch64 Linux, but the packaged app build script currently validates only release/Scenespy-linux-x64/Scenespy.
- ARM64 Linux should be treated as unsupported until a separate ARM64 app build is produced and tested.

Suggested release note:
Linux build tested on the latest Ubuntu x86_64 release. Other x86_64 glibc-based distributions may work, but are not yet certified. FFmpeg installation requires apt, dnf, pacman, or an existing ffmpeg/ffprobe in PATH. Detect faces uses CPU unless a supported NVIDIA CUDA setup is detected.
